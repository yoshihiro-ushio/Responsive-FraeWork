動画の直接埋め込み

推奨記述。
controls属性あり、preload属性はmetadata、poster属性（サムネ画像）あり、ダウンロード禁止、複数フォーマット対応 

				HTML
				
<video controls preload="metadata" poster="http://placehold.it/1280x720?text=Thmbnail">
	<source src="../src/movie/sample.mp4" type="video/mp4">
	<source src="../src/movie/sample_end.webm" type="video/webm">
	<source src="../src/movie/sample_end.ogg" type="video/ogg">
	※現在の環境は動画再生に対応していません
</video>

基本的な記述

						HTML
						
<video src="[ファイルパス]"></video>
  • video要素の中は空に（video要素非対応だとそのまま表示される）
  • デフォルトではコントロールパネルは非表示（表示させないとブラウザによっては再生できない）
  • ブラウザによっては、コントロールパネルを表示させないと動画自体表示されない（iPhone Safariなど）
  • 要素の特性は基本的にimg要素と似ているので、動画サイズがコンテンツ幅より大きい場合ははみ出す（サンプルはmax-width: 100%;が付与されています）

コントロールパネルの表示

						HTML
						
<video controls src="[ファイルパス]"></video>
  • controls属性を追加（値は不要）
  • コントロールパネルのデザインは、ブラウザや再生メディア（動画・音のみ）によって変わる場合がある
  • 右クリックで制御の上書き可能（ブラウザによるかも）

コントロールパネルの表示内容を制御

						HTML
						
<video controls controlslist="nodownload nofullscreen noremoteplayback"></video>
  • controlslist="nodownload"：ダウンロードボタン非表示
  • controlslist="nofullscreen"：フルスクリーン非表示
  • controlslist="noremoteplayback"：リモート再生機能禁止？？
  • この方法でも右クリックからの命令は禁止できない（ただコントロールパネルから消えるだけ）

nodownload

nofullscreen

nodownload + nofullscreen

ループ再生

						HTML
						
<video controls loop src="[ファイルパス]"></video>
  • loop属性を追加（値は不要）
  • 右クリックで制御の上書き可能（ブラウザによるかも）

ミュート（消音）

						HTML
						
<video controls muted src="[ファイルパス]"></video>
  • muted属性を追加（値は不要）
  • コントロールパネルの音量ボタンで変更可能

自動再生

						HTML
						
<video controls muted autoplay src="[ファイルパス]"></video>
  • loop属性を追加（値は不要）
  • 一部ブラウザでは、ミュート（muted属性）を同時に付与しないと自動再生されない（UX観点と思われる）
  • ブラウザによっては、ミュート指定してても自動再生しない場合もある（？）

サムネイル設定

						HTML
						
<video controls src="[ファイルパス]" poster="[サムネ画像パス]"></video>
  • poster属性にファイルパスを記述
  • サムネ画像を設定しないと、動画の0秒が表示、もしくは何も表示されないので、設定推奨。

動画の事前読み込み

						HTML
						
<video controls preload="metadata" src="[ファイルパス]"></video>
  • preload="none"：ファイルを事前に読み込まない
  • preload="auto"：ファイルを事前に読み込む
  • preload="metadata"：ファイルのメタデータ（長さなど）だけ事前に読み込む
  • loop属性を追加（値は不要）
  • 一部ブラウザでは、ミュート（muted属性）を同時に付与しないと自動再生されない（UX観点と思われる）
  • ブラウザによっては、ミュート指定してても自動再生しない場合もある（？）

auto

metadata

ピクチャーインピクチャー禁止

「ピクチャーインピクチャー」とは、動画再生意中にスクロールなどで動画が消えても、画面端に動画が表示され続ける、ブラウザの機能。 

						HTML
						
<video controls disablePictureInPicture></video>
  • disablePictureInPicture属性を記述（値は不要）
  • コントロールパネルから消えるだけで完全な禁止ではない（？）

複数のファイルを用意する

複数のファイルを連続で再生・・・ということではなく、古い一部のブラウザでは【mp4】に対応していない場合があり、その代替ファイルを複数用意する場合の記述。
上から順に優先され、1番目が非対応なら2番目・・・と優先される。
video要素そのものが対応していない場合は、video要素内の記述がそのまま表示される。 

						HTML
						
<video controls>
	<source src="[ファイルパス]" type="[ファイルタイプ]">
	<source src="[ファイルパス]" type="[ファイルタイプ]">
	<source src="[ファイルパス]" type="[ファイルタイプ]">
	※現在の環境は動画再生に対応していません
</video>
  • source要素のtype属性を指定しないとファイルが読み込まれない可能性あり
  • type属性の記述は「video/拡張子」（mp4なら「type="video/mp4"」）
  • source要素は先（上）優先で読み込まれる
  • video要素の中は、video要素対応ブラウザでは動画以外表示されず、非対応の場合はそのままの内容が表示される
